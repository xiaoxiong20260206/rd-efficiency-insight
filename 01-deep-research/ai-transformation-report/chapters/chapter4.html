应该做什么：从"人人会用"到"工种重塑"

阶段一解决了"能不能用起来"的问题。阶段二要解决的是——每个工种如何与AI深度协同，让工作方式发生质变而非量变。这是最容易被误解的阶段：很多企业以为"大家都在用AI"就算转型了，但PwC 2026报告尖锐指出——

"Instead of cutting a few steps, rethink the workflow, which an AI-first approach may turn into a single step. That often starts by asking not how AI can fit into a workflow but how it can create a new one."

—— PwC, 2026 AI Business Predictions

阶段二的三件事

1. 工种工作流重设计——不是"AI辅助旧流程"，而是"AI原生新流程"
2. 分工种AI成熟度评估——用L1/L2/L3模型定位每个工种的AI化水平
3. Agentic工作流落地——让Agent接管重复性专业任务

1.1 工种工作流重设计：问"如何从零设计"，而非"AI放在哪里"

回到智能手机类比：每个人都有了智能手机，摄影师开始用手机拍照、记者用手机写稿——每个工种的工作方式被重新定义，但团队协作模式还没变。同理，如果一个PM还是按“写PRD → 评审 → 排期 → 跟进”的流程工作，只是用AI“加速写PRD”，那效率提升可能只有20-30%。

但如果从头设计一个AI原生的PM工作流呢？

环节 传统流程 AI辅助（阶段一） AI原生（阶段二）
需求发现 用户访谈 + 竞品调研 AI辅助写调研问卷 Agent自动抓取用户反馈、竞品动态，生成需求洞察
需求定义 PM独立撰写PRD AI帮写PRD初稿 Agent基于需求洞察直接生成PRD + 原型 + 测试用例
进度管理 每日站会 + 手动更新 AI总结会议纪要 Agent实时同步进度，异常自动预警和干预建议
效果验证 上线后手动分析 AI辅助数据分析 Agent自动追踪指标，生成归因分析和迭代建议

BCG 2026报告指出：在AI原生模式下，一个PM的覆盖范围可以扩大4-6倍。这不是"PM用AI快了30%"，而是"PM的工作本质变了"——从"执行者"变成"指挥官"。

1.2 分工种AI成熟度评估：L1/L2/L3模型

不同工种的AI化路径和速度不同。以下三级模型可以帮助定位每个工种当前在什么水平、下一步应该做什么：

成熟度 定义 特征 典型场景
L1 AI辅助 AI作为效率工具 人主导，AI加速；工作流不变 AI写初稿、AI翻译、AI代码补全
L2 AI协同 AI参与关键决策 人+AI协作，工作流重设计 AI生成方案 + 人审核决策、Agent自动执行子流程
L3 AI自主 AI独立完成子任务 Agent端到端执行，人只做监督和异常处理 Agent自动化测试、Agent自动运维、Agent自动客服

1.3 Agentic工作流落地：PwC的"Spell It Out"原则

PwC 2026报告给出了一个关键方法论：在设计Agentic工作流时，必须逐步骤写清楚——哪些步骤由Agent主导，哪些由人主导，哪些是人机协作，以及每一步的监督机制。

PwC "Spell It Out" 工作流设计框架

对于每个关键工作流中的每一步，明确回答四个问题：

  • 谁主导？ Agent独立执行 / 人独立执行 / 人机协作
  • 谁监督？ 人工Review / 另一个Agent交叉检查 / 自动化测试
  • 何时升级？ 什么条件下需要人工介入
  • 如何回滚？ 出错时的恢复机制

"Built-in monitoring also includes different agents checking each other's work, and for higher-risk scenarios, these agents come from different model providers."

—— PwC, 2026 AI Business Predictions

具体怎么做：分工种AI化路线图

阶段二不是"全员齐步走"，而是按工种分批推进。不同工种的AI化难度和ROI不同，应该优先攻克高ROI、低复杂度的工种。

2.1 典型工种AI化路径

工种 当前普遍水平 阶段二目标 关键变化 参考实践
研发工程师 L1 代码补全 L2 AI协同开发 从"AI补全代码"到"AI生成模块 + 人审核 + AI测试" Cursor + Agentic Coding
测试工程师 L1 AI辅助写用例 L3 Agent自动化测试 从"AI帮写测试脚本"到"Agent自动生成 + 执行 + 回归 + 报告" AI Test Agent
产品经理 L1 AI辅助写文档 L2 AI协同产品设计 从"AI写PRD"到"AI洞察需求 + 生成方案 + 模拟验证" 覆盖范围扩大4-6倍
设计师 L1 AI生图 L2 AI协同设计 从"AI出草图"到"AI生成多方案 + 人选定 + AI出终稿 + 切图" Figma AI + MidJourney
运营/市场 L1 AI写文案 L2 AI个性化运营 从"AI写通稿"到"Agent分人群生成个性化内容 + 自动分发 + 效果追踪" PwC: hyper-personalization
财务/HR L1 AI辅助分析 L3 Agent自动执行 Agent处理发票、匹配订单、对账、异常检测；人聚焦战略决策 PwC: AI agents for finance

2.2 落地六步法

步骤 动作 交付物 周期
Step 1 选择3-5个高ROI工种作为试点 优先级矩阵（ROI × 复杂度） W1-W2
Step 2 对每个工种做"一日工作流审计"——记录每个步骤及耗时 当前工作流地图 W2-W3
Step 3 设计AI原生工作流——从零思考"如果AI是默认的，怎么设计" 目标工作流地图 + 人机分工矩阵 W3-W5
Step 4 构建必要的Agent/自动化流程 可用的Agentic工作流 W5-W10
Step 5 试点团队试跑2-4周，收集数据 效率对比数据 + 问题清单 W10-W14
Step 6 迭代优化后推广到全工种 标准化AI工作流手册 W14-W24
Deloitte的关键提醒：别忘了度量标准的升级

Deloitte 2026报告指出，当工种AI化后，传统的度量标准会失效。例如：用"代码行数"衡量AI辅助开发是荒谬的。新的度量标准应该聚焦业务产出而非工作量——比如从"一周写了多少行代码"变成"一周交付了几个可用Feature"。

3

能带来哪些变化

阶段二完成后，你应该能观察到这些变化：

维度 变化前（阶段一完成） 变化后（阶段二完成） 衡量指标
工作模式 AI是"加速器" AI是"协作者" 关键工种Agent覆盖率 > 60%
交付效率 个人效率提升20-30% 关键工种交付效率翻倍 交付周期缩短 > 50%
工作内容 重复性工作占比高 重复性工作大部分由Agent完成 人工重复任务占比 < 30%
度量体系 基于"工作量" 基于"业务产出" 每个工种有AI时代的新KPI
能力要求 专业技能 + 基本AI使用 专业技能 + Agent编排能力 核心岗位AI进阶培训通过率 > 80%
阶段二做对的信号
  • 员工开始说"这件事没有Agent就太低效了"
  • 工种之间出现新的协作模式
  • 新入职员工觉得旧流程"不可思议"
  • 度量标准从输入端转向输出端
阶段二的典型陷阱
  • "AI使用率很高"但交付效率没变（阶段一假象）
  • 工作流没变，只是每个步骤"AI加速了一点"
  • Agent建了很多但没有实际业务价值（demo陷阱）
  • 度量标准还在用传统指标衡量新模式
阶段二的限度——诚实面对

阶段二完成后，你不应该期望看到：

  • 团队规模显著缩减（那需要阶段三的结构重组）
  • 工种之间的壁垒消除（那需要阶段三的协作模式重塑）
  • 全新的业务模式出现（那需要阶段四的战略跃迁）

阶段二的成功标准是"让每个工种找到自己的AI原生工作方式"——这是从局部优化到整体变革的关键跳板。
